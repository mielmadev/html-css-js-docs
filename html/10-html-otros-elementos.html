10. Otros elementos

Dentro de la especificación HTML existen más elementos para proporcionar valor semántico al contenido, por ejemplo:

Iframe

El elemento iframe se utiliza para insertar contenido de otra página web dentro de una página existente, por ejemplo, un video de Youtube, un mapa de Google Maps...

Por lo general, nosotros no vamos a crear este elemento. Cuando vayamos a insertar este contenido, la página donde se encuentra ese contenido a insertar nos facilitará el código para copiar y pegar, que no podremos modificar.

El contenido dentro del iframe es completamente independiente del contenido de nuestra página, por lo que nuestros estilos CSS y funciones Javascript no se aplicarán al contenido dentro del iframe. 


<!-- Video Youtube -->
<iframe width="560" height="315" src="https://www.youtube-nocookie.com/embed/pxUSYa3u7rs?si=mC4gi8Z2YPkzV2Ad" title="YouTube video player" frameborder="0" allow="accelerometer; autoplay; clipboard-write; encrypted-media; gyroscope; picture-in-picture; web-share" allowfullscreen></iframe>

<!-- Mapa Google Maps -->

Dirección

El elemento address se utiliza para marcar la información de contacto del autor/responsable de la página (ya sea una dirección postal, un teléfono, un e-mail...). y sólo debe incluir esta información específica. Generalmente lo vamos a utilizar en el pie de página (que es donde suele ubicarse) 


<address>
	<p>Empresa ACME<br>
	C/ Gran Vía nº 1<br>
	48000 Bilbao (Vizcaya)<br>
	Teléfono: 94 444 444 444
	</p>
</address>

No debe utilizarse para representar direcciones de manera arbitraria. Por ejemplo, si tuviéramos que implementar un directorio de empresas similar a esta web, el elemento address sólo se implementaría en el pie de página, con la dirección postal del periódico. Las direcciones postales/teléfonos de las empresas se implementarían simplemente con párrafo.

Superíndice y Subíndice

El elemento sup se aplica a un texto que se tiene que mostrar como superíndice, esto es, más alto y más pequeño que el texto principal, como es el caso de números y letras en notaciones científicas, matemáticas o para referencias en un texto.

El elemento sub por el contrario, se aplica a un texto que se tiene que mostrar como subíndice:más bajo y pequeño. 


<p>La parcela de mide 20m<sup>2</sup></p>

<p>La fórmula química de agua es: H<sub>2</sub>O</p>

Estas etiquetas no deben usarse para lograr un efecto visual sin más, para ello utilizaremos CSS.

Elemento b

El elemento b representa un fragmento de texto que se resalta por ser importante en relación al contenido: palabras clave de un documento, nombres de productos en una revisión del mismo, o cualquier otro fragmento de texto que se represente en negrita en modo escrito. 


<h2>Opiniones sobre Caracoles Lentos</h2>
<p>
	Los <b class="nom-product">Caracoles Lentos</b>
	son un nuevo producto de la empresa alimenticia....
</p>

Elemento i

El elemento i representa un fragmento de texto en voz alternativa o contenido que se presenta con itálica (cursiva) en modo escrito: palabras en otro idioma (utilizando el atributo lang), términos técnicos y taxonómicos, notación musical, pensamientos, cambios en el estado de ánimo, referencia a contenido escrito a mano...

Es recomendable utilizar clases (atributo class) para indicar el significado que se le quiere dar al elemento i en cada caso (de este modo se demuestra que no se está utilizando como elemento de diseño): 


<p>
	Los <i lang="ca">escargots</i>, o caracoles,
	son de la familia de los <i class="taxonomy">Helix aspersa</i>.
</p>

Tecla

El elemento kbd lo utilizaremos para indicar que el texto mostrado pertenece a una tecla o combinación de teclas que tendremos que introducir por nuestro teclado. 


<p>Pulsa <kbd>F5</kbd> para refrescar la página.</p>

Mediante CSS se puede simular una tecla física de teclado, como se puede ver en el capítulo 4 de la Introducción del Curso.

Insertado y borrado

A veces, cuando modificamos un dato de una página web (p.e. una fecha, precio en rebajas, etc.), nos interesa dejar constancia de esa modificación. Para ello vamos a utilizar los elementos ins y del.

del se utiliza para marcar el contenido antiguo e ins, para marcar el contenido nuevo que lo ha sustituído. 


<p>El sorteo se realizará el 2 de abril a las <del>11</del> <ins>12</ins> de la mañana.</p>

Preformateo y código

El elemento pre representa texto preformateado, es decir, se respetarán los espacios, saltos de línea y tabulaciones tal y como están escritos: 


<pre>
	function saludar() {
		console.log('¡Hola, clase!');
	}
</pre>

El elemento code sirve para indicar que el texto que estamos mostrando pertenece a un código informático y normalmente suele utilizarse junto con pre para mantener los saltos de línea, sangrías, etc.: 


<code>
	class HelloWorld {
		public static void main(String[] args) {
			System.out.println("Hello World!");
		}
	}
</code>

Orden de tabulador

El atributo tabindex permite modificar el orden de navegación mediante el tabulador. El valor de este atributo será numérico, indicando el orden en caso de haber más de uno. Si la información de la página está bien estructurada y en orden, es muy raro que necesitemos este atributo.

En ocasiones se utiliza para parchear algo que se ha planteado mal y la solución resultante suele ser aún peor.

Por ejemplo, imaginemos que tenemos una página con enlaces de navegación, contenido y un formulario para suscribirse a una newsletter. Al entrar en la página, el foco se coloca automáticamente en el campo de formulario, pasando por alto los enlaces de navegación y cualquier otro elemento interactivo que pueda haber antes.

See the Pen HTML - Tabindex (mal uso) by Antxoa (@antxoa) on CodePen.

Otro ejemplo podría ser un formulario (mal) maquetado con tablas. Como el orden de navegación sigue siempre la secuencia del código fuente, la secuencia será: nombre, población, apellidos, cp, dirección y teléfono, ya que las tablas se construyen por filas (y no por columnas).

Cuando el usuario vaya a rellenar el formulario, si está acostumbrado a pasar de un campo a otro con tabulador del teclado, se le hará muy raro el orden de relleno de los campos, ya que lo normal es introducir el nombre, después los apellidos, después de la población el cp, etc... 


<table>
	<tr>
		<td>Nombre:<br /><input type="text" name="nom" tabindex="1" /></td>
		<td>Población:<br /><input type="text" name="pob" tabindex="4" /></td>
	</tr>
	<tr>
		<td>Apellidos:<br /><input type="text" name="apel" tabindex="2" /></td>
		<td>CP:<br /><input type="text" name="cp" tabindex="5" /></td>
	</tr>
	<tr>
		<td>Dirección:<br /><input type="text" name="dir" tabindex="3" /></td>
		<td>Teléfono:<br /><input type="text" name="tfn" tabindex="6" /></td>
	</tr>
</table>

Nota: este código de formulario en incorrecto en muchos sentidos (semántico, accesible) y sólo se utiliza para ilustrar un mal uso del atributo tabindex, no debe tomarse como base para usar en una web real.

Acceso directo

El atributo accesskey permite implementar atajos de teclado para los enlaces más utilizados en nuestra web (al igual que los atajos de teclado en un programa software).

Basta con utilizar el atributo con un valor (tecla) de letra o número: 


<a href="index.html" accesskey="1">Página de inicio</a>

La combinación de teclas varía de unos navegadores a otros:

Windows Linux Mac
Firefox Alt + Shift + tecla Control + Option + tecla

o Control + Alt + tecla
MS Edge Alt + tecla Control + Option + tecla

o Control + Option + Shift + tecla 		Control + Option + tecla
Google Chrome
Safari n/a Control + Option + tecla
Opera Alt + tecla Control + Alt + tecla

Es recomendable utilizar sólo los números para evitar posibles interferencias con las letras que sirven como atajo de teclado en los navegadores y ayudas técnicas. También se recomienda utilizar el número "1" para la página de inicio.

Chrome suele ignorar estos accesos directos.
