19. Elementos extra

Color

Si necesitamos un campo para poder elegir o insertar directamente el código hexadecimal de un color, utilizaremos: 


<p>
	<label for="colorPpal">Color principal</label>
	<input type="color" name="colorPpal" id="colorPpal">
</p>

Datalist

El campo datalist es un híbrido entre un campo select y otro de texto, color, fecha... De inicio se visualiza un campo de texto donde podemos escribir un dato, y mientras escribimos nos muestra un listado de sugerencias, de tal manera que podemos optar por elegir una opción de entre las sugeridas o bien escribir el dato: 


<p>
	<label for="campoMusicos">Músico favorito:</label>
	<input type="text" name="campoMusicos" id="campoMusicos" list="listaMusicos">
	<datalist id="listaMusicos">
		
		
		
		
		
		
		
		
		
	</datalist>
</p>

<p>
	<label for="campoHora">Reserva hora:</label>
	<input type="time" name="campoHora" id="campoHora" list="listaHoras">
	<datalist id="listaHoras">
		
		
		
	</datalist>
</p>

También es posible visualizar una etiqueta asociada a cada valor:


<p>
	<label for="campoTto">Tratamiento:</label>
	<input type="text" name="campoTto" id="campoTto" list="listaTratamiento">
	<datalist id="listaTratamiento">

Campo oculto

En ocasiones necesitamos campos de formulario que no están visibles públicamente, pero que sí que están a nivel operativo en el formulario, para añadir por programación cierta información. Estos campos ocultos no llevan un elemento label asociado puesto que el usuario no lo va a ver ni escuchar, ni tampoco modificar su valor: 


<input type="hidden" name="codigo" id="codigo" value="a445832">

Output

El campo output se utiliza para representar el resultado de una operación (por ejemplo, una multiplicación de 2 campos, cálculo de edad, etc... mediante javascript). Este campo no lleva ningún label asociado puesto que el usuario no interactúa con él:

En el ejemplo del rango visto anteriormente se utiliza este campo para mostrar el valor del rango en cada momento. 


<output></output>

fieldset / legend

El elemento fieldset se utiliza para categorizar, diferenciar y agrupar campos de formulario y los propios formularios.

Existen tres casos donde podemos utilizar este elemento. A modo de curiosidad, el validador de HTML del W3C usa este elemento para los casos 1 y 3.

Caso 1: diferenciar datos

En un formulario podemos tener datos de diferentes tipos (por ejemplo, datos personales y datos bancarios) y para diferenciarlos de una forma lógica podemos usar el elemento fieldset para agrupar cada tipo.

Para indicar el rótulo/título de cada tipo de datos utilizaremos el elemento legend:

See the Pen HTML - Fieldset (uso nº1) by Antxoa (@antxoa) on CodePen. 


<form>
	<fieldset>
		<legend>Datos personales</legend>
		<p>
			<label for="nombre">Nombre:</label>
			<input type="text" name="nombre" id="nombre">
		</p>
		<p>
			<label for="e-mail">E-mail</label>
			<input type="text" name="e-mail" id="e-mail">
		</p>
	</fieldset>

	<fieldset>
		<legend>Datos bancarios</legend>
		<p>
			<label for="banco">Banco:</label>
			<input type="text" name="banco" id="banco"><br>
		</p>
		<p>
			<label for="numcuenta">Número de cuenta:</label>
			<input type="text" name="numcuenta " id="numcuenta">
		</p>
	</fieldset>

	<p><input type="submit" value="enviar"></p>
</form>

Caso 3: diferenciar formularios

Si en la página tenemos más de un formulario (por ejemplo, formulario de contacto y formulario de búsqueda) podemos utilizar un elemento fieldset por cada formulario: 


<fieldset>
	Formulario de búsqueda
	

		... campos del formulario ...
	

</fieldset>

<fieldset>
	Formulario de contacto
	

		... campos del formulario ...
	

</fieldset>

search

El elemento search se utiliza para marcar cualquier sección de una página web dedicada a la búsqueda, ya sea un formulario de entrada, una barra de búsqueda, instrucciones y ejemplos de la búsqueda...

Sería similar al elemento fieldset pero únicamente incluyendo un formulario de búsqueda / resultados. 


<search>
	

		
		
		
	

</search>
