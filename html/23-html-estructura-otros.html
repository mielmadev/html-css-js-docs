23. Otros elementos estructurales

progress / meter

Estos dos elementos son realmente campos de formulario, pero el usuario no puede introducir información en ellos ya que los valores los controlamos desde el desarrollo (principalmente con Javascript).

progress

El elemento progress se utiliza para indicar el progreso o avance de una operación o tarea, de tal manera que el usuario pueda saber cuánto falta para terminar la tarea, en qué estado está...

Visualmente se representa con una barra visual que va mostrando este progreso.

Se puede utilizar en diferentes casos: estado de la carga de un video/animación, avance en la finalización de un temario, encuesta, proceso de compra...

See the Pen HTML - Progress by Antxoa (@antxoa) on CodePen.

meter

El elemento meter indica el nivel o escala de un cierto valor, por ejemplo, cuánto de segura es tu contraseña, nivel de batería, espacio libre en Gmail, cuántos conocimientos tenemos sobre un tema, cuántas personas han acudido a votar, cuántas entradas se han vendido...

No se debe utilizar para medir cosas de las que no conocemos el valor máximo. Por ejemplo, si vamos a indicar que un árbol mide 5 metros, esa información no sería un elemento meter, ya que no podemos saber cuál es la altura máxima de un árbol, para establecer la relación entre la altura de un árbol concreto con respecto al más alto que exista.

See the Pen HTML - Meter by Antxoa (@antxoa) on CodePen.

time

El elemento time se utiliza para marcar contenido que representa una fecha, hora, duración, o un punto específico en el tiempo de manera semántica.

Permite asociar un valor legible por humanos-usuarios con un valor legible por máquinas, a través del atributo datetime en un formato estandarizado (ISO 8601). 


<time datetime="2024-01-25">25/01/2024</time>
<time datetime="2024-01-25">25 de Enero de 2024</time>
<time datetime="2024-01-25">25 - Enero - 2024</time>
<time datetime="2024-01-25">Veinticinco de Enero de dos mil veinticuatro</time>
El plazo de inscripción terminó <time datetime="2024-03-19">ayer</time>

No debemos usar time en los siguientes casos:

Texto no temporal:
Si el texto no representa una fecha, hora o duración específica (p.e. "el próximo verano" sin una fecha exacta)
Contenido decorativo:
Si la fecha/hora es solo parte del diseño y no tiene valor semántico
Fecha imprecisa:
Si no se puede indicar el valor exacto en datetime (p. e. "siglo XX")

details / summary

El elemento details nos permite implementar bloques de contenido para que se muestren y oculten a petición del usuario. Estos bloques se llaman comúnmente bloques acordeón o bloques desplegables.

Inicialmente estos bloques estarán ocultos, mostrándose únicamente su título y un icono representativo (símbolo +, flecha...). Al pulsar en el título, el bloque se desplegará y si volvemos a pulsar, se contraerá y ocultará.

Atributos personalizados

La especificación HTML permite crear atributos personalizados (para luego utilizarlos con Javascript, CSS...) siempre y cuando los nombres de estos atributos empiecen por el prefijo data-.

Si no empiezan con este prefijo, al validar el código, nos dará error. 


 <p id="dir1" data-provincia="Bizkaia" data-cp="48001" data-color="#FF0000">Bilbao</p>
 <p id="dir2" data-provincia="Gipuzkoa" data-cp="20010" data-color="#0000FF">Donosti</p>

dialog

El elemento dialog se utiliza para crear ventanas de diálogo o modales que muestran contenido destacado, como alertas, mensajes, confirmaciones o información adicional que requiere la atención del usuario.

Puede operar de dos formas:

Modal:
Bloquea la interacción con el resto de la página hasta que se cierra, con un fondo oscurecido (backdrop).
No modal:
Permite interactuar con el contenido de fondo mientras el diálogo está visible.

El elemento está oculto por defecto. Para que se muestre es necesario aplicarle el atributo open (con Javascript).

Aviso

¡Este es un mensaje importante!

 

<button id="openButton">Abrir diálogo</button>
<dialog id="myDialog">
	<h3>Aviso</h3>
	<p>¡Este es un mensaje importante!</p>
	<button id="closeButton">Cerrar</button>
</dialog>

popovers

El atributo popover es un atributo global que convierte cualquier elemento en una ventana emergente (popover).

Estas ventanas aparecen superpuestas al contenido principal de la página y se utilizan para mostrar información temporal, como tooltips, menús contextuales, notificaciones o paneles informativos. No requieren bloquear la interacción con el resto de la página.

Es una solución nativa para crear elementos emergentes sin depender de JavaScript o bibliotecas externas.

Este es un tooltip simple.

Primero tenemos que identificar el bloque con el contenido emergente a través de un id y utilizar el atributo popover con 2 posibles valores.

popover="auto"
El contenido emergente se cierra automáticamente al hacer clic fuera del elemento o al presionar la tecla Esc.
popover="manual"
El contenido emergente permanece abierto hasta que se cierra explícitamente.

Después, en el botón que abrirá este contenido emergente, añadimos el atributo popovertarget con el valor del id anterior, para poderlos relacionar entre sí. 


<button popovertarget="mitooltip">Ver información</button>
<div id="mitooltip" popover="auto">
	<p>Este es un tooltip simple.</p>
</div>

También existe el atributo popovertargetaction que define la acción a realizar si queremos un control personalizado. Los valores de este atributo son: show | hide | toggle.

  • Opción 1
  • Opción 2

<button popovertarget="menu">Abrir menú</button>
<div id="menu" popover="manual">
	<ul>
		<li><a href="#">Opción 1</a></li>
		<li><a href="#">Opción 2</a></li>
		<li><button popovertarget="menu" popovertargetaction="hide">Cerrar</button></li>
		</ul>
</div>

La apariencia gráfica de este elemento se puede modificar mediante CSS.

hr

El elemento hr (horizontal rule) tiene el aspecto de una línea horizontal, pero no es un simple separador visual.

Semánticamente representa una ruptura temática entre párrafos de una sección o entre secciones de contenido, es decir, indica un cambio de tema o de contexto dentro de un mismo flujo narrativo.

Podremos modificar su aspecto visual mediante CSS más adelante.


<article>
  <h1>Detrás del rodaje de "Película Inventada"</h1>
  <p>El rodaje de "Película Inventada" estuvo lleno de dificultades técnicas y logísticas.</p>
  <p>Las escenas de acción en exteriores requirieron cerrar calles y coordinar a cientos de extras.</p>
  <hr>
  <p>Tras finalizar la producción, la película tuvo una distribución internacional limitada. </p>
  <p>Solo se proyectó en algunos cines de Europa y América del Norte, y se lanzó directamente en plataformas de streaming en otros países.</p>
  <hr>
  <p>A pesar de sus esfuerzos, la película recibió varios premios Razzie por peor guion y peor dirección. </p>
  <p>Sin embargo, algunos críticos valoraron positivamente los efectos visuales y la banda sonora.</p>
</article>

El primer hr marca la transición entre dificultades del rodaje y distribución de la película.

El segundo hr indica el cambio hacia premios y críticas.

div / span

Los elementos div y span son elementos que carecen de significado semántico. Se utilizan para agrupar código de tal manera que luego podamos aplicar estilos a través de CSS.

Cuando estemos realizando la estructura HTML de una página web, no vamos a utilizar ninguno de estos dos elementos. En el momento que pasemos a maquetar con CSS, se verá si necesitamos utilizarlos y será el momento de añadirlos. 


<div>
	....
</div>
<span>....</span>

Cuando no existían las etiquetas semánticas de estructura, utilizábamos div continuamente. Para diferenciar un div de otro, usábamos atributos id y class.

Ahora reemplazamos muchos de esos div por sus etiquetas semánticas apropiadas. Seguiremos teniendo div, pero sólo para agrupar elementos con el objetido de aplicarles CSS.

Divitis

La Divitis es un término coloquial en desarrollo web que describe el uso excesivo o indebido de la etiqueta div para todo en una página HTML, sin aprovechar los elementos semánticos adecuados.

Efectos negativos

Pérdida de semántica:
Muchas div no dicen nada sobre su contenido; al no usar elementos semánticos como header, nav, article, section, etc., se pierde la estructura lógica del documento.
Accesibilidad degradada:
Los lectores de pantalla y tecnologías de asistencia tienen dificultades para interpretar contenido muy anidado sin semántica clara.
Mantenimiento difícil:
Muchos div anidados generan un DOM complejo, dificultando localizar, depurar o modificar estilos y comportamientos.
SEO y rendimiento:
El marcado con muchos contenedores innecesarios añade peso, complejidad y reduce la claridad para los motores de búsqueda.

Cómo evitarlo

Usar elementos semánticos:
Usar header, nav, main, article, section, aside, footer cuando corresponda.
Evitar div innecesarios:
No sustituir con div lo que ya representa un elemento semántico, por ejemplo, usar ul + li para listas en vez de div anidados.
Reducir anidamiento:
Si un contenedor no aporta estructura semántica, probablemente no sea necesario.
Mantener marcado limpio:
Escribir HTML claro, legible y fácil de mantener.
Usar clases/id sobre elementos semánticos:
Para estilos y maquetado, no reemplazar estructura con div salvo necesidad específica.
