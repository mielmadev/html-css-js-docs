5. Imágenes

Sintaxis básica

Las imágenes son elementos que se implementan mediante la etiqueta <img>. Esta etiqueta no va a tener etiqueta de cierre, sólo de apertura.

Necesitaremos 2 atributos de caracter obligatorio que son src para la ruta del archivo de imagen y alt para el texto alternativo de la imagen: 


<img src="ruta" alt="texto alternativo">

incorrecto:
<img>
<img src="ruta">
<img alt="ACME">

Si la imagen que vamos a enlazar se encuentra en la misma carpeta que la página en la que estamos, bastará con escribir el nombre de la imagen.

Si está alojada en una subcarpeta, entonces utilizamos una barra / para indicar el camino y si se encuentra en una carpeta de un nivel superior, utilizamos dos puntos y una barra ../ para subir de nivel: 


<img src="ruta" alt="texto alternativo">

<img src="logo.jpg" alt="ACME">

<img src="imagenes/logo.jpg" alt="ACME">

<img src="../logo.jpg" alt="ACME">

incorrecto:
<img>
<img src="logo.jpg">
<img alt="ACME">

Podemos subir o bajar de nivel las veces que sea necesario:


<img src="imagenes/minis/logos/logo.jpg" alt="ACME">

<img src="../../../logos/logo.jpg" alt="ACME">

Texto alternativo

El atributo alt es obligatorio y es necesario ponerlo en el código. Otra historia será el valor que pongamos en dicho atributo.

Este atributo sirve para ofrecer un texto equivalente al contenido/función de la imagen en caso de que ésta no se cargue o bien un usuario navegue sin ver la pantalla, mediante un lector de pantalla, o un buscador indexando la página (Google, por ejemplo). Hay toda una ciencia detrás de cuál tiene que ser el mejor texto a usar como texto alternativo en una imagen y de ello se ocupa el estándar de Accesibilidad Web.

Si estamos en un entorno visual y podemos ver la pantalla, el texto alternativo será transparente para nosotros, no lo vamos a ver nunca

A modo general, vamos a establecer las siguientes recomendaciones:

  • Si la imagen está dentro de un enlace, el texto alternativo se corresponderá con el texto que pondríamos en el enlace si lo hiciéramos sin imagen.
  • Si la imagen ofrece información importante, el texto alternativo tendrá que ofrecer la misma información (p.e. un símbolo de lavado de ropa).
  • Si la imagen es decorativa, esto es, no aporta ningún tipo de información y si no se llegase a cargar no perderíamos ninguna información, el texto alternativo lo dejaremos vacío. 

<img src="lavado-mano.jpg" alt="Lavado a mano">
<img src="personas.jpg" alt="">

<a href="https://www.google.es">
	<img src="logo-google.jpg" alt="Google"></a>
</a>

alt y title

En algunas comunidades de desarrolladores, está muy extendida la práctica de añadir el atributo title a las imágenes, con el mismo valor que el del atributo alt. Esto viene de un error heredado muchos años atrás. En su día ocurría lo siguiente:

Como hemos dicho, el atributo alt tiene un uso específico, para personas o máquinas que no ven la pantalla. El difunto navegador Internet Explorer, como no respetaba los estándares HTML, decidió que iba a tratar este atributo como el atributo title, y al pasar con el ratón por encima de una imagen que tuviera alt, sacaba el texto alternativo igual que nos aparecen los atributos title. El resto de navegadores no lo hacían así, porque seguían correctamente el estándar.

Algunos desarrolladores se dieron cuenta de estas diferencias de visualización y empezaron a añadir a las imágenes el atributo title con el mismo valor, para que así, independientemente del navegador con el que visitáramos la web, nos aparecería un texto al pasar por la imagen. Esto se fue generalizando porque "todo el mundo lo hace".

Cada atributo tiene su propia función y no los podemos utilizar para otros menesteres, y en este caso, cada atributo tiene una función diferente.

En la práctica es muy raro que una imagen necesite un atributo title, sólo si se está muy seguro de su uso y existe una justificación adecuada se debe utilizar.

Formato de imagen a utilizar

Los formatos a utilizar son los generales para imágenes: PNG, JPG (JPEG), GIF, SVG, WEBP, AVIF...

Mapas de Imágenes

Un mapa de imágenes es simplemente una imagen sobre la que se definen unas áreas activas (hotspots) que actuarán como enlaces. Si tomamos una imagen de un mapa del mundo, por ejemplo, cada continente podría ser un área activa.

En la actualidad apenas se utilizan y de hacerlo, usaremos un generador y nos aseguraremos de que el resultado se adapte a cualquier dispositivo, tamaño de ventana, etc.
