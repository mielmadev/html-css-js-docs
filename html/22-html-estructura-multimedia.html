22. Multimedia

figure / figcaption

El elemento figure permite asociar contenido embebido (diagramas, ilustraciones, fotos, video, audio...)

Lleva conjuntamente el elemento figcaption que se utilizará para implementar el texto asociado al contenido embebido, a modo de pie de foto. 


<figure>
	<img src="/imagenes/grafica-vida.jpg" alt="Gráfica">
	<figcaption>
		Gráfica 1: Vida de un desarrollador.
	</figcaption>
</figure>

No debemos utilizar figure de forma indiscriminada, sino en los siguientes casos:

Es auto-contenido:
El contenido dentro de figure debe tener sentido por sí mismo, incluso si se extrae del contexto del documento principal.
Complementa el contenido principal:
No es esencial para entender el texto principal, pero lo enriquece (por ejemplo, una imagen ilustrativa, un gráfico de apoyo o una cita relevante).
Puede ser referenciado:
Es un contenido al que se puede hacer referencia en el texto (por ejemplo, "ver Figura 1") o que se beneficiaría de una descripción en un figcaption.
Incluye una descripción opcional:
Si el contenido necesita un título, crédito, o explicación (por ejemplo, "Gráfico de ventas 2024"), se incluye en un figcaption.

picture / srcset

El elemento picture, junto con el atributo srcset tienen como objetivo dar una solución a los problemas de adaptación de una imagen a los distintos dispositivos (tamaños, calidades...), permitiendo utilizar varias imágenes de las cuales sólo será visible una de ellas (la que sea más óptima en cada dispositivo).

Explicaremos su funcionamiento en el Módulo de RWD. 


<picture>
	<source media="(min-width: 1000px)" srcset="/imagenes/kitten-large.png">
	<source media="(min-width: 600px)" srcset="/imagenes/kitten-medium.png">
	<img src="/imagenes/kitten-small.png" alt="Gatito">
</picture>

Video

Mediante el elemento video y una serie de atributos, podemos insertar un video de manera nativa:

src
La URL del contenido de audio/video (puede reemplazarse por el atributo source).
autoplay (true|false):
Indica si el archivo debe reproducirse automáticamente o no.
loop (true|false):
Indica si el archivo debe volverse a reproducir una vez llegado a su fin.
muted (true|false):
Indica si el sonido debe estar activado inicialmente o no. Si no se indica nada, el sonido estará activado.
preload (none|metadata|auto):
Indica al navegador cierta información del archivo.
La diferencia entre metadata y auto es que metadata sólo pre-carga la información del archivo (p.e. la duración que va a tener) y con auto se cargará todo el archivo (independientemente de que el usuario lo reproduzca o no).
controls:
Indica si se deben mostrar los controles de reproducción o no.
source:
Debido a las diferencias de soporte de formatos entre los diferentes navegadores(mp4, webm, ogg), es recomendable ofrecer el contenido en diferentes formatos. Para indicarle al navegador cuáles son, prescindiremos del atributo src (ya que sólo puede enlazar con un único archivo) y lo reemplazaremos por varias instancias del atributo source (tantas como formatos dispongamos).
poster:
Si el video tarda en cargarse (o no lo hace por alguna razón), podemos establecer una imagen estática (un fotograma del video, por ejemplo) que se muestre en ese caso. 

<video src="video-publicidad.webm" autoplay="false" loop="true" muted="true" preload="auto" controls poster="caratula-video.png"></video>

<video controls>
	
	
	
</video>

Pistas

El elemento track permite ofrecer subtítulos, títulos, descripciones, capítulos o metadatos para el elemento multimedia.

La información textual se implementa en un archivo separado del elemento multimedia, que puede estar en diferentes formatos (siempre y cuando sean formatos soportado por los navegadores), aunque es recomendable usar el éstándar que ofrece el W3C, el formato WebVTT.

Controles personalizados:

Es posible programar y diseñar nuestros propios controles personalizados para el reproductor de audio/video. Para ello, será necesario programarlo con Javascript, utilizando los atributos y métodos habilitados para tal efecto, o bien utilizar alguna librería ya creada.

Audio

En el elemento audio vamos a utilizar los mismos atributos que en el elemento video a excepción de la carátula y las pistas. 


<audio src="entrevista.mp3" controls></audio>
