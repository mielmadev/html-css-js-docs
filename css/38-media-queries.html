38. Media Queries

Debido a la multitud de dispositivos desde los que accedemos a Internet, las posibles dimensiones de la pantalla del dispositivo así como sus capacidades se han disparado. De ahí que haya nacido el concepto de Responsive Web Design.

CSS permite comprobar diferentes valores del dispositivo, como la anchura y altura tanto del dispositivo como de la pantalla y el tipo de color de la pantalla (color, monocromo...), entre otras.

Estas comprobaciones son lo que llamamos Media queries (en singular, Media Query) y funcionan como condiciones que el navegador evalúa para determinar si tiene que aplicar unos estilos o no. Por ejemplo, podemos indicar que se aplique un fondo de un color cuando la pantalla es menor de un tamaño determinado y que se aplique otro color diferente si la pantalla es mayor de ese tamaño.

Las condiciones se pueden utilizar directamente en la llamada del archivo css, por ejemplo:


<link rel="stylesheet" type="text/css" href="estilos.css" media="screen and (max-width: 980px)">

O en el propio código CSS, que será la forma habitual:


@media screen and (max-width: 980px) {
	Código CSS
	...
}

Indicaremos también en qué dispositivos se tiene que aplicar la condición, puesto que podemos crear un código css para un dispositivo de pantalla, impresora, proyección, etc.

Además es necesario indicar explícitamente a los navegadores que estamos utilizando Media queries para que nos apliquen las condiciones, añadiendo en la cabecera de la página la siguiente línea:


<meta name="viewport" content="width=device-width, initial-scale=1.0">

Las Media queries por sí mismas no hacen que una página tenga automáticamente un RWD. En el módulo específico de Responsive Web Design aprenderemos a utilizarlas en toda su extensión.
