16. Márgenes

Existen dos tipos de márgenes: los internos y los externos:

Margen interno (padding)

Un elemento tiene márgenes internos por cada lado. Este espacio se denomina padding y se encuentra entre el contenido del elemento y su borde.

Podemos establecer el mismo padding para los 4 lados, o bien los 4 valores individualmente e incluso escribir los 4 en una misma propiedad unificada. En este caso, los valores se separarán por espacios y orden tiene que ser en sentido de las agujas del reloj (empezando a las 12 - arriba):

Márgenes colapsados:

Cuando dos elementos tienen márgenes externos superior y/o inferior, la mayoría de veces estos márgenes se fusionan entre sí y el margen mayor engulle al pequeño (en vez de sumar ambos márgenes). Es lo que se conoce como márgenes colapsados.

En los márgenes externos laterales esto no ocurre.

Para que los márgenes superiores e inferiores se colapsen, se tienen que dar los siguientes supuestos:

Caso 1: elementos adyacentes

En este caso, como el margen del elemento 1 es mayor que el margen del elemento 2, el margen ganador será el del elemento 1.

Caso 2: elementos hermanos no adyacentes

Si los dos elementos no están adyacentes, es decir, entre medias hay otro elemento, pero este elemento está vacío (no tiene contenido), el navegador ignora a este elemento vacío y colapsa los márgenes, como si fueran adyacentes.

Si el elemento entre medias tiene contenido, entonces no colapsa los márgenes.

En el caso excepcional del elemento br, que es un elemento vacío por naturaleza, si es el que se encuentra entre medias, el navegador lo interpreta como un elemento con contenido y no colapsa los márgenes.

Caso 3: elementos hermanos y sobrino

Si tenemos varios elementos hermanos y uno de ellos lo englobamos dentro de otro tipo de elementos (pasa a ser sobrino de los que eran sus hermanos), el navegador ignora al padre de este elemento, los considera a todos hermanos y por lo tanto, colapsa los márgenes.

Si el elemento padre tiene un altura establecida, o padding o un borde, entonces ya no ocurre esto.

Caso 4: padre e hijo con márgenes en la misma dirección

Si el padre tiene un margen superior y el hijo también tiene un margen superior, ambos se colapsan. Si tienen los dos un margen inferior, también.

Caso 5: hermanos con el mismo tipo de hijos

Si tenemos dos elementos hermanos y ambos tienen el mismo tipo de hijo, los márgenes también se colapsan (se ignora a los padres).

Caso 6: múltiples márgenes colapsados

Se puede dar el caso de tener varios elementos consecutivos y englobados y que aún así se colapsen todos sus márgenes.

Caso 7: márgenes negativos y positivos

En el caso de que haya márgenes negativos, sólos o en combinación con márgenes positivos, se realiza la suma/resta de los valores para determinar el valor final del margen.
