27. Normalización y Reseteo

Introducción

Así como el cambio del Modelo de Caja es imprescindible en cualquier maquetación, también es altamente recomendable utilizar una serie de estilos predeterminados, tanto para igualar los estilos estándar de los elementos entre navegadores como para establecer unos estilos de uso frecuente.

Normalización

Para igual los estilos predeterminados que los navegadores aplican a los elementos podemos normalizar las propiedades CSS que aplican.

Por ejemplo, ante una lista desordenada ul, algunos navegadores añaden un margen lateral izquierdo en el propio ul para simular la sangría, mientras que otros navegadores añaden ese margen en el elemento li. Visualmente es imposible saber en qué elemento está aplicando el margen el navegador.

Como no es viable probar todos los elementos en todos los navegadores y versiones e implementar esta normalización partiendo desde cero, vamos a utilizar una CSS que recoge todos estos cambios entre navegadores y los normaliza.

Reseteo

Hay desarrolladores que, además de utilizar la normalización, quieren empezar con todos los elementos sin ningún tipo de estilo, reseteándolos, para ir añadiendo los necesarios a demanda.

Personalización

Lo más recomendable y habitual es utilizar los estilos de normalización, algunos (o todos) los de reseto y todos ellos juntarlos con estilos CSS que vamos a querer utilizar siempre en nuestros proyectos: modelo de caja alterado, quitar subrayado a los enlaces, cambio de cursores en algunos elementos, comportamientos de scroll... De esta manera podremos empezar a maquetar nuestra página sabiendo que nos vamos a ahorrar tiempo y pruebas.

A medida que vayamos haciendo proyectos, podremos ir averiguando cuáles vamos a utilizar siempre de inicio, por lo que nuestra CSS inicial será flexible e irá cambiando a lo largo del tiempo.
