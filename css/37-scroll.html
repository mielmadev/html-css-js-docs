Barras de scroll en CSS

Scroll

Estilos de las barras de scroll

Si necesitamos modificar los colores de las barras de scroll, vamos a utilizar un código estándar y otro con prefijo -webkit-.

Primero establecemos dos colores, uno para el fondo de la barra de scroll y otro para la barra propiamente dicha. Después establecemos la anchura de la barra: 


:root{
	--color-fondo: #deeafc;		/* Azul claro */
	--color-barra: #006eba;		/* Azul oscuro */
}


*{
  scrollbar-width: thin;			/* valores auto | thin */
  scrollbar-color: var(--color-barra) var(--color-fondo);
}


/* Para Safari y Navegadores móviles (excepto Firefox) */

*::-webkit-scrollbar {
  width: 12px;
}

*::-webkit-scrollbar-track {
  background: var(--color-fondo);
}

*::-webkit-scrollbar-thumb {
	background-color: var(--color-barra);
	border-radius:0;
	border:0;
}

Aunque en el código con prefijos es posible establecer una anchura determinada y unos bordes a la barra de scroll, esto no forma parte de la especificación estándar y cuando los navegadores adopten al 100% la especificación, eliminarán esas opcione que no son estándar.

Barras de scroll en un elemento concreto

Si queremos aplicar los estilos de las barras a un único elemento, entonces llamamos al selector del elemento, en vez de al selector universal: 


elemento{
  scrollbar-width: thin;
  scrollbar-color: var(--color-barra) var(--color-fondo);
}

Comportamiento del scroll

Cuando ejecutamos un enlace dentro de la misma página, el movimiento automático del scroll no es brusco, sino que tiene una "animación".

Históricamente este efecto se realizaba con Javascript pero a día de hoy es posible implementarlo desde CSS con la propiedad scroll-behavior aplicada al elemento html: 


html{scroll-behavior: smooth;}	/* Hay animación */

html{scroll-behavior: auto;}		/* No hay animación, el scroll es brusco */

Separación al hacer scroll

En muchas ocasiones, cuando ejecutamos un enlace dentro de la página, nos damos cuenta que el scroll no se posiciona en el elemento sino que se posiciona unos cuantos pixels antes o después.

Esto provoca un efecto molesto ya que el usuario tiene que corregir la posición del scroll para poder ver el contenido correctamente (sobre todo cuando la cabecera de página se mantiene fija).

Para corregir la posición del scroll, hasta hace relativamente poco tiempo teníamos que utilizar Javascript, pero por fin tenemos una solución 100% en CSS, mediante la propiedad scroll-padding-top: 


html{scroll-padding-top: 60px;}

El valor que indiquemos será la separación que se añadirá a la posición del scroll.

Además podemos modificar también el padding inferior y los dos laterales, si fuera necesario.
