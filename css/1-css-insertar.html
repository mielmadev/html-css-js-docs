Insertar CSS

Existen 4 métodos para insertar código CSS en nuestra página Web:

Estilos externos

También llamados estilos enlazados, es el método más utilizado. Crearemos uno o varios archivos CSS donde estarán todas las propiedades que vamos a utilizar y desde nuestra página HTML llamaremos al archivo (o archivos) CSS en el elemento head.

El elemento que sirve para incluir el archivo de estilos es link y va a necesitar 4 atributos: 

<link rel="stylesheet" href="ruta/estilos.css" type="text/css" media="all">

Atributo href

Indica la ruta al archivo de estilos (igual que si estamos creando un enlace a otro archivo html). Los archivos a enlazar tendrán extensión .css y el nombre deberá cumplir la misma normativa que para los archivos .html (ver normativa).

Atributo rel

Este atributo establece una relación de tipo de hoja de estilo entre el archivo html y el archivo css. Siempre será stylesheet.

Atributo type

Este atributo indica que el contenido del archivo css es texto y con código css (se utiliza el MIME type adecuado para los archivos .css).

Atributo media

El atributo media puede llevar diferentes valores, en función del dispositivo en el que queremos que se cargue el archivo de estilos:

media="all"
Para todos los dispositivos
media="screen"
Para dispositivos de pantalla
media="print"
Para dispositivos de impresión

Si omitimos el atributo media, los dispositivos asumen que el archivo se va a cargar siempre, para todos (es decir, sería como usar un valor all): 

<link rel="stylesheet" href="ruta/estilos.css" type="text/css">

Es posible cargar el archivo en varios dispositivos, separando los valores con comas: 

<link rel="stylesheet" href="ruta/estilos.css" type="text/css" media="screen,print">

Estilos embebidos

Este método se utiliza principalmente para insertar estilos únicos de la página/sección en la que nos encontramos. También se insertará en el elemento head de nuestra página HTML: 

<style type="text/css" media="all">
  a{text-decoration:none;}
</style>

Estilos en línea

En este método, insertamos las propiedades CSS mediante el atributo style en nuestro archivo HTML, por lo que sólo se aplicará a ese elemento concreto: 

<a href="index.html" style="text-decoration:none">Inicio</a>

Este método sólo se debe utilizar en los siguientes dos casos:

  • Mientras estamos desarrollando una página web, haciendo pruebas, en una formación..., pero nunca en la web publicada.
  • Si en nuestra página web existe un editor WYSIWYG (como TinyMCE, CKEditor o similares) y por lo tanto no tenemos ningún control sobre el contenido que va a generar el usuario, como ocurre en gestores de contenido (CMS) como Wordpress, Magento...

En cualquier otro caso, no se debe utilizar este método.

El motivo es que este método dificulta mucho el mantenimiento de la web, ya que si queremos cambiar una propiedad que hemos aplicado a varios elementos, tendremos que buscar y modificar cada uno de ellos.

Además, este método tiene la máxima prioridad, por lo que si en un futuro queremos cambiar esa propiedad desde una hoja de estilos externa o embebida, no podremos hacerlo a menos que utilicemos !important, lo que complicará aún más el mantenimiento de la web.

La regla @import

La regla @import permite importar hojas de estilo desde otras hojas de estilo. Se puede usar de las siguientes formas: 

@import "ruta/hoja_de_estilo.css";
@import url("ruta/hoja_de_estilo.css");

Esta declaración deberá ir antes que cualquier otra regla en la hoja de estilo.

Cuando se utiliza este método, el navegador primero carga todo el código HTML de forma secuencial, saltándose los estilos. Y hasta que no haya cargado completamente la página web no es cuando descarga y aplica los estilos. Es por ello que primero veremos la página sin estilos y después de un tiempo (medio segundo, varios segundos...) se cargarán.

A este comportamiento se le denomina FUOC: Flash of Unstyled Content (también puede ocurrir al implementar determinadas librerías javascript en diferido) y por lo general es bastante molesto.

Por lo tanto, este método está desaconsejado salvo que sea necesario por requerimientos específicos y se sepa muy bien el por qué se va a utilizar.
