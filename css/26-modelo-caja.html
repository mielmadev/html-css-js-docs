26. Modelo de Caja

Modelo de Caja Estándar

En CSS cada elemento tiene un área de contenido y rodeándolo, opcionalmente, áreas de márgenes (margin y padding) y bordes.

El ancho total del elemento viene dado por la suma de los márgenes y bordes (izquierdos y derechos), y el ancho del contenido. La altura está dada por la suma de los márgenes y bordes (superiores e inferiores), y la altura del contenido:

Por lo tanto si tenemos un elemento con las siguientes propiedades:

div {
width: 100px;
padding: 10px;
border: 5px solid black;
margin: 20px;
}

Aunque creamos que nuestro bloque tiene una anchura de 100px, en realidad, sumándole el margin, padding y border de cada lado, tiene una anchura de 170px:

170px = 100px + 10px + 20px + 10px + 20px + 5px + 5px

Este es el comportamiento estándar del modelo de caja definido en la especificación CSS.

Modelo de Caja Alterado

Un poco de historia

En el pasado, algunos navegadores (p.e. IE 5.5) no respetaban el comportamiento del Modelo de Caja y utilizaban un Modelo de Caja erróneo. En vez de sumar los valores de los márgenes y el borde a la anchura de un elemento, lo restaban.

Eso provocaba que teníamos que utilizar diferentes hacks u otras técnicas de maquetación para que el modelo de caja se aplicase correctamente en todos los navegadores.

Si analizamos el modelo de caja estándar desde un punto de vista práctico, a la hora de realizar la maquetación es complicado de aplicar, ya que tenemos que estar continuamente calculando el tamaño final del elemento. Y ese modelo alterado que se utilizaba en el pasado resultaría mucho más práctico y sencillo.

Ahora

En la actualidad contamos con la propiedad box-sizing que permite cambiar el modelo de caja estándar por un modelo alterado, similar al erróneo, pero con alguna variación:

  • La anchura del elemento será una anchura en bruto.
  • Si se añaden padding y/o border, ocuparán espacio dentro del elemento: nos restan espacio para el contenido pero la anchura en bruto se mantiene.
  • Si se añade margin, se sumará a la anchura del elemento.

Una vez que los navegadores actuales dieron soporte a esta propiedad, hubo un antes y un después en cuanto a maquetación se refiere, puesto que este cambio de modelo ha permitido maquetar páginas de forma más sencilla y sin necesidad de hacer cálculos extra, ahorrándonos muchas horas de depuración y quebraderos de cabeza.

Para poder empezar a maquetar sin problemas, tenemos que insertar el siguiente código en nuestra hoja de estilos para que se cambie el modelo de caja en todos los elementos:

html {
box-sizing: border-box;
}
*, *:before, *:after {
box-sizing: inherit;
}
/* Nota importante: 
   Este bloque debería estar en el HTML inicial de todos los proyectos.
   Así aseguras un modelo de caja consistente y evitas problemas de maquetación.
*/
Comparación de valores de box-sizing
Valor Descripción ¿Incluye padding y border en el ancho?
content-box Modelo estándar. El ancho y alto solo afectan al contenido. No
border-box Modelo alterado. El ancho y alto incluyen el padding y el borde. Sí

Y de esta manera, todos los elementos HTML heredarán esta propiedad y podremos olvidarnos de hacer cálculos.

Si después quisiéramos que algún elemento concreto volviera a utilizar el Modelo de Caja tradicional, entonces utilizaríamos el siguiente valor:

elemento {
box-sizing: content-box;
}
