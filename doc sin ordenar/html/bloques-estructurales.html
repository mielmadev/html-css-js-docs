Bloques estructurales

Para estructurar de forma semántica los diferentes bloques estructurales de una página web (cabecera, pie, contenido principal, secciones...) disponemos de varios elementos.

Header

El elemento header representa un grupo introductorio de información y donde se incluirá un encabezado (h1-h6). También puede usarse para agrupar la tabla de contenidos de una sección, logotipos relevantes, ayudas de navegación, etc. Pueden existir varios header en una página (en función del contenido). 


  <header>
    Aquí va la cabecera de la página siempre tiene que tener algún encabezado.
  </header>
Aquí va la cabecera de la página siempre tiene que tener algún encabezado.

Footer

El elemento footer representa el pie de la sección de contenido predecesora más cercana, o la sección del contenido raíz. Normalmente contiene información acerca de su sección, como el autor, enlaces a documentos relacionados, licencias, apéndices, índices, etc. También es posible incluir la información que habitualmente se incluye en el elemento address (información de contacto del autor o editor, por ejemplo). El elemento footer no tiene por qué aparecer en el final de la sección. Pueden existir varios footer en una página. 


  <footer>
    Aquí va el pie de página.
  </footer>
Aquí va el pie de página.

Main

El elemento main representa la parte de la página donde se encuentra el contenido principal de la misma. Debe ser un contenido único en el documento y excluye el contenido que se repite en la web como menús de navegación, copyright, logotipo, banners, formulario de búsqueda (salvo que estemos en la página específica de búsqueda). No puede haber más de un elemento main visible en la página. El elemento main sólo puede tener como ancestros a body, div y form (este último, bajo ciertas circunstancias). 


  <header>
    Header: Aquí va la cabecera de la página.
  </header>

  <main>
    Main: Aquí va el contenido principal de la página.
  </main>

  <footer>
    Footer: Aquí va el pie de página.
  </footer>

Nav

El elemento nav representa una sección de la página que contiene enlaces de navegación a zonas de la misma página o a otras páginas. No todos los grupos de enlaces tienen que implementarse con el elemento nav. Sólo se implementarán los bloques principales de navegación. Por ejemplo, el menú principal de la web sí que se incluiría en un elemento nav, pero un listado de enlaces a recursos externos, no.

Aside

El elemento aside representa una sección de la página cuyo contenido está relacionado tangencialmente con el contenido que tiene a su alrededor, pero se considera contenido independiente, adicional. Se puede usar para implementar barras laterales, publicidad, elementos con efectos tipográficos (como las citas). No es adecuado utilizarlo para estructurar contenido explicativo, porque forma parte del contenido principal. 


  <aside>
    Aquí va una barra lateral con enlaces a artículos relacionados o publicidad.
  </aside>
Aquí va una barra lateral con enlaces a artículos relacionados o publicidad.

Article

El elemento article representa una composición autónoma (contenido único) en un documento, página,aplicación o sitio web, con intención de poder ser reutilizado (por ejemplo, en un lector RSS, al imprimir la página, visualización de página simplificada en dispositivo móvil...). Puede utilizarse en un artículo de un foro, revista, artículo de periódico, entrada (post) de un blog, comentario escrito por un usuario, widget interactivo o gadget e incluso en cualquier otro elemento independiente de contenido. Cuando existen elementos article anidados, los elementos article interiores estarían relacionados con los article exteriores (por ejemplo, un article que contenga un artículo de un blog, contendrá también article para los comentarios de un usuario).

Section

El elemento section representa una sección genérica de un documento o aplicación. Una sección, en este contexto, es una agrupación temática de contenido, habitualmente con un encabezado. El elemento section no es un elemento contenedor genérico. Si sólo se necesita el elemento para aplicar estilos, entonces se deberá utilizar el elemento div. El elemento section puede contener más elementos section. 


  <section>
    Aquí va una sección de contenido.
  </section>
Aquí va una sección de contenido.

Media

Imágenes con figure y figcaption

El elemento figure permite asociar contenido embebido (diagramas, ilustraciones, fotos, video, audio...) con un texto. Preferiblemente debe hacerse una referencia en el texto de la página hacia el bloque figure. El elemento figcaption se utiliza para implementar el texto asociado al contenido embebido, a modo de pie de foto. 


  <figure>
    <img src="img/grafica-vida.jpg" alt="Gráfica">
    <figcaption>
      Gráfica 1: Vida de un desarrollador.
    </figcaption>
  </figure>

Imágenes adaptativas con picture y srcset

El elemento picture, junto con el atributo srcset, permite adaptar una imagen a los distintos dispositivos (tamaños, calidades...), permitiendo utilizar varias imágenes de las cuales sólo será visible una de ellas (la que sea más óptima en cada dispositivo). 


  <picture>
    <source media="(min-width: 1000px)" srcset="img/kitten-large.png">
    <source media="(min-width: 600px)" srcset="img/kitten-medium.png">
    <img src="img/kitten-small.png" alt="Gatito">
  </picture>

Video

Mediante el elemento video y una serie de atributos, podemos insertar un video de manera nativa. Los atributos más comunes son:

  • src: La URL del contenido de audio/video (puede reemplazarse por el atributo source).
  • autoplay (true|false): Indica si el archivo debe reproducirse automáticamente o no.
  • loop (true|false): Indica si el archivo debe volverse a reproducir una vez llegado a su fin.
  • muted (true|false): Indica si el sonido debe estar activado inicialmente o no.
  • preload (none|metadata|auto): Indica al navegador cierta información del archivo.
  • controls: Indica si se deben mostrar los controles de reproducción o no.
  • source: Permite ofrecer el contenido en diferentes formatos.
  • poster: Imagen estática que se muestra mientras el video carga.

  <video src="video-publicidad.webm" autoplay="false" loop="true" muted="true" preload="auto" controls
    poster="caratula-video.png"></video>

Es recomendable ofrecer el contenido en diferentes formatos para asegurar la compatibilidad entre navegadores. 


  <video controls>
    <source src="video-sorteo.mp4">
    <source src="video-sorteo.webm">
    <source src="video-sorteo.ogg">
  </video>

Pistas y subtítulos en video

El elemento track permite ofrecer subtítulos, títulos, descripciones, capítulos o metadatos para el elemento multimedia. La información textual se implementa en un archivo separado, generalmente en formato WebVTT. 


  <video controls>
    <source src="video.mp4">
    <track src="subtitulos.vtt" kind="subtitles" srclang="es" label="Español">
  </video>

Controles personalizados

Es posible programar y diseñar controles personalizados para el reproductor de audio/video usando JavaScript y los métodos y atributos del elemento multimedia, o bien utilizar alguna librería ya creada.

Audio

El elemento audio permite insertar archivos de sonido en la página. Se utilizan los mismos atributos que en el elemento video, a excepción de la carátula y las pistas. 


  <audio src="entrevista.mp3" controls></audio>

Fieldset y Legend

El elemento fieldset se utiliza para categorizar, diferenciar y agrupar campos de formulario y los propios formularios. El elemento legend se usa para indicar el rótulo o título de cada grupo de campos. 


  <fieldset>
    <legend>Formulario de búsqueda</legend>
    <form name="formBusqueda">
      ... campos del formulario ...
    </form>
  </fieldset>

  <fieldset>
    <legend>Formulario de contacto</legend>
    <form name="formContacto">
      ... campos del formulario ...
    </form>
  </fieldset>

Progress

El elemento progress se utiliza para indicar el progreso o avance de una operación o tarea, de tal manera que el usuario pueda saber cuánto falta para terminar la tarea o en qué estado está. Visualmente se representa con una barra visual que va mostrando este progreso. 


  <progress value="70" max="100">70%</progress>
70%

Meter

El elemento meter indica el nivel o escala de un cierto valor, por ejemplo, cuánto de segura es tu contraseña, espacio libre en un servicio, cuántos conocimientos tenemos sobre un tema, etc. No se debe utilizar para medir cosas de las que no conocemos el valor máximo. 


  <meter value="0.6">60%</meter>
60%

Time

El elemento time permite indicar una fecha en un formato legible para máquinas, facilitando la interpretación de fechas por parte de sistemas automáticos.

Details y Summary

El elemento details permite implementar bloques de contenido que se muestran y ocultan a petición del usuario, conocidos como bloques acordeón o desplegables. El elemento summary define el título visible del bloque. 


  <details>
    <summary>Más información</summary>
    Este es el contenido adicional que se puede mostrar u ocultar.
  </details>

Div y Span

Los elementos div y span son elementos sin significado semántico. Se utilizan para agrupar código y aplicar estilos a través de CSS. Se recomienda usarlos solo cuando sea necesario para maquetación o estilos. 


  <div>
    ....
  </div>
