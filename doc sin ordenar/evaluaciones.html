Evaluaciones A lo largo del curso vamos a tener unas evaluaciones individuales por cada módulo y una evaluación final de curso. Tanto en las evaluaciones individuales como en la evaluación final se realizará un examen tipo test de 10 preguntas y una prueba técnica. La prueba técnica varía entre módulos y evaluación final: Evaluación de cada módulo La prueba técnica de cada módulo consistirá en revisar una web ya hecha, detectar los fallos de código que pueda haber y corregirlos. Para ello seguiremos el siguiente proceso: Detectar fallos de cada módulo correspondiente (HTML, CSS, JS, Usabilidad y Accesibilidad), utilizando los validadores de HTML/CSS, inspector del navegador, herramientas propias de Accesibilidad y técnicas heurísticas de Usabilidad. Documentar detalladamente estos fallos en un documento. En el caso de Accesibilidad, indicar además el criterio WCAG que se incumple. Indicar en el documento la corrección que se va a realizar. Finalmente, modificar el código fuente para obtener una web libre de errores. Envío de correcciones Una vez completadas todas las correcciones y el informe, es necesario enviar el documento a través del campus virtual. En las evaluaciones de módulo sólo hay que enviar el informe, no el código fuente. Corrección a realizar: Modificar el enlace por uno correcto Evaluación final de curso La prueba técnica consiste en crear una página web de ejemplo de CV del alumno en base al diseño gráfico propuesto. Se adjuntará una carpeta con los archivos HTML/CSS base. Estos archivos se modificarán añadiendo el código necesario, así como añadiendo otros archivos para javascript, tipografías, videos y cualquier otro recursos que se quiera utilizar. También se adjuntan las imágenes que aparecen en el diseño pero se pueden reemplazar por cualquier otra (que deberán tener una licencia de uso gratuita). El alumno es libre de utilizar la combinación de colores y tipografías que quiera (no tienen que ser los del diseño), siempre y cuando cumpla con los criterios de accesiibilidad. La Web deberá cumplir los requisitos que se indican a continuación: UF1841: HTML y CSS El código fuente tiene que estar libre de errores HTML/CSS. La estructura HTML tendrá que ser la adecuada, utilizando las etiquetas necesarias para cada elemento. Los textos pueden ser Lorem Ipsum, inventados o reales. Será necesario utilizar al menos una tipografía personalizada (bien de Google Fonts -recomendable-, otro repositorio o tipografía propia). Se implementarán transiciones en los enlaces y botones. Los enlaces del menú deberán llevar a los diferentes apartados de la página. La cabecera debe permanecer fija en la parte superior al hacer scroll. El contenido mínimo debe ser el indicado en el diseño gráfico. Si el alumno quiere incluir más contenido/apartados, se evaluará como una mejora adicional. UF1842: Javascript Deben incluirse 2 funcionalidades Javascript como mínimo en la página. Estas funciones pueden ser personalizadas o bien pueden ser librerías externas incluidas en la web. Por ejemplo: Ampliación de las imágenes de Portfolio. Reemplazar las imágenes de portfolio y su diseño a 4 columnas para sustituirlas por una galería de imágenes que van rotando, etc... Modificar de manera aleatoria la imagen de fondo de cabecera, o los colores de las secciones, o cualquier otro elemento. Crear una validación personalizada con JS en el formulario de contacto. Añadir un reloj con la fecha actual. Añadir una cuenta atrás para un acontecimiento determinado. ... : UF1843: Accesibilidad y Usabilidad La web tiene que cumplir con los criterios de accesibilidad AA. Para ello será necesario revisar todos los puntos y hacer especial hincapié en textos alternativos, combinación de colores, estructura, funcionalidad...
