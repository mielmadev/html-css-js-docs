33. Grid (parte 1)

Introducción

Grid es un sistema con un conjunto de propiedades creadas ex profeso para maquetar los contenidos en 2 dimensiones (filas y columnas). En una maquetación lo más habitual suele ser utilizar Flexbox y Grid a la vez, puesto que cada uno tiene sus usos específicos. Las principales diferencias entre Flexbox y Grid son las siguientes: Una dimensión contra dos: Flexbox está pensado para maquetar en una dimensión (eje principal) y Grid para 2 dimensiones (filas y columnas). Basado en contenido o en diseño: Flexbox se basa en el contenido mientras que Grid se basa en el diseño. Esfuerzo de aprendizaje: Flexbox es relativamente sencillo de aprender y aplicar. La especificación de Grid es bastante compleja ya tienen muchas propiedades (tanto para el contenedor padre como para los hijos) y dispone de varios métodos para lograr una misma maquetación.

Definir columnas

grid-template-columns

La propiedad grid-template-columns indica explícitamente el número de columnas que va a tener nuestra maquetación, así como la anchura de cada una de ellas. Podemos mezclar diferentes unidades de medida, como px, %, rem, vw, la nueva unidad fr (fracción), la palabra reservada auto y otras funciones para establecer el tamaño. En esta primera parte vamos a ver las unidades de medida más básicas. grid-template-columns: 150px 150px 150px; grid-template-columns: repeat(3, 150px); 1º segundo bloque tercero Por defecto los bloques se alinean en la parte izquierda y además todos se igualan en altura (en función del bloque más alto). Estas alineaciones se podrán modificar más adelante. grid-template-columns: 300px 120px 120px; grid-template-columns: 300px repeat(2, 120px); 1º segundo bloque tercero grid-template-columns: 33% 33% 33%; grid-template-columns: repeat(3, 33%); 1º segundo bloque tercero La unidad de medida fr (fracción) es específica de Grid y representa una parte del espacio libre que queda en una fila y/o columna. Es el equivalente a la propiedad flex-grow que utilizamos en Flexbox: grid-template-columns: 1fr 1fr 1fr; grid-template-columns: repeat(3, 1fr); 1º segundo bloque tercero grid-template-columns: 1fr 2fr 1fr; 1º segundo bloque tercero grid-template-columns: 3fr 1fr; 1º segundo bloque Se pueden mezclar todas las unidades de medida, combinándolas a nuestro antojo. Por ejemplo, vamos a dejar una columna siempre fija (su tamaño no variará), otra relativa al tamaño del padre y otra en base al tamaño libre que quede: grid-template-columns: 150px 30% 1fr; 1º segundo bloque tercero También podemos utilizar la palabra auto, para que el navegador establezca la anchura automática: grid-template-columns: auto auto auto; grid-template-columns: repeat(3, auto); 1º segundo bloque tercero A veces da la sensación que auto hace exactamente lo mismo que fr, pero no es así. El valor fr calcula el espacio libre y auto establece una anchura automática. Si no se combina con otras unidades de medida, reparte todo el espacio entre los elementos (en este caso sí que funciona como fr). Pero si se combina con otras unidades de medida, la anchura de los elementos vendrá dada por la anchura del contenido de cada elemento: grid-template-columns: auto 1fr auto; 1º segundo bloque tercero

Filas implícitas (automáticas)

Si hay más elementos que columnas, éstos se van distribuyendo automáticamente en filas, respetando el tamaño que tenga cada columna. Es lo que llamamos filas implícitas. En el caso de la altura, cada fila será independiente y por defecto tomará la altura máxima de los elementos por cada fila (posteriormente se podrá modificar): grid-template-columns: 1fr 2fr 1fr;

Definir filas explícitas

grid-template-rows

La propiedad grid-template-rows indica el número de filas que va a tener nuestra maquetación, así como la altura de cada una de ellas. También podemos mezclar diferentes unidades de medida, hacer filas fijas y otras que ocupen automáticamente el espacio sobrante, etc. Si queremos maquetar varios elementos en multi-línea, todas las líneas iguales, pero no sabemos cuántas filas puede haber (porque dependerá del nº de elementos que haya), no necesitamos establecer el número de líneas de antemano. Simplemente se establecen las columnas (como hemos visto en el último ejemplo del apartado de Columnas) y las filas se van añadiendo según sea necesario. Si tenemos que hacer una maquetación con un número de filas y columnas concretas, entonces sí que combinaremos esta propiedad con grid-template-columns, salvo que queramos hacer una maquetación específica en la que sólo haya una única columna. Al no especificar explícitamente la columna, existirá de manera implícita: grid-template-rows: 150px 150px 150px; grid-template-rows: repeat(3, 150px); 1º segundo bloque tercero Muchas de las veces no queremos establecer una altura concreta a la fila, queremos que el contenido fluya y que ocupe la altura que necesite. Entonces utilizaremos la palabra auto: grid-template-rows: auto auto auto; grid-template-rows: repeat(3, auto); 1º segundo bloque tercero Y podemos combinar múltiples valores: grid-template-rows:80px auto auto; grid-template-rows:80px repeat(2, auto); 1º segundo bloque tercero Si establecemos un número determinado de filas pero hay más elementos, se crean nuevas filas implícitamente: grid-template-rows:80px auto 80px; 1º segundo bloque tercero

Combinación de filas y columnas

Para establecer tanto filas como columnas de manera explicita, basta con combinar las dos propiedades anteriores: grid-template-columns: 2fr 1fr 1fr; grid-template-rows: 100px auto; 1º segundo bloque tercero 4º 5º sexto Si hay más bloques de los establecidos inicialmente, se crea una nueva fila implícitamente con altura automática y respetando las anchuras de las columnas: grid-template-columns: 2fr 1fr 1fr; grid-template-rows: 100px auto; 1º segundo bloque tercero 4º 5º sexto 7º octavo 9%

Separación entre filas y columnas

Podemos indicar una misma separación entre filas y entre columnas: grid-template-columns: repeat(3, 1fr); gap: 50px; 1º segundo bloque tercero 4º 5º sexto Establecer valores diferentes para la filas y para las columnas: grid-template-columns: repeat(3, 1fr); gap: 20px 50px; [fila] [columna] 1º segundo bloque tercero 4º 5º sexto O establecer los valores en propiedades individuales: grid-template-columns: repeat(3, 1fr); row-gap: 20px; column-gap:50px; 1º segundo bloque tercero 4º 5º sexto Si las columnas y/o filas tienen el tamaño expresado en una unidad de medida diferente a fr o auto, al añadir la separación entre filas/columnas, este tamaño de separación se suma al tamaño de los elementos, por lo que el contenido se desborda: grid-template-columns: repeat(3, 33%); gap: 50px; 1º segundo bloque tercero 4º 5º sexto

Propiedades al hijo

Líneas de ubicación

Se puede usar para maquetar toda la página directamente, colocando el header, footer, main, publicidad..... donde se quiera.

      grid-template-columns: 1fr;  única columna
      grid-template-rows: auto auto 1fr auto; Filas automáticas y una que ocupe el espacio sobrante

el grid son tablas con lineas, empezando desde la linea uno al principio de la tabla, la 2 entre la uno y dos, la dos, entre las dos y la tres y la cuatro después de la tres

para que una celda ocupe toda la linea o las celdas que nosotros queramos, se modifican los hijos.

      
        .item:nth-child(2){}
        .item:nth-child(3){}
        .item:nth-child(4){}
        .item:nth-child(5){}

        .item:nth-child(6){
          grid-row-start: 3;
          grid-row-end: 4;
          grid-column-start: 1;
          grid-column-end: 4;
        }
      
      El sexto hijo (.item:nth-child(6)) se coloca en la tercera fila (de la línea 3 a la 4)
      y ocupa todas las columnas (de la columna 1 a la 4), es decir, se expande a lo largo de toda la fila 3.
.celda-padre { display: grid; grid-template-columns: 1fr 2fr 1fr; grid-template-rows: auto; } 
      Colocar el hijo en la celda correspondiente:
      
        .celda-hijo {
          grid-column-start: 1;
          grid-column-end: 2;
          grid-row-start: 1;
          grid-row-end: 2;
        }
.item:nth-child(1) { grid-column-start: 1; grid-column-end: 2; grid-row-start: 1; grid-row-end: 2; }

se pueden elegir solo bien horizontales o verticales, no se puede hacer una ele para elegir las filas, siempre tiene que ser cuadrado o rectangular

se puede abreviar: grid-column: 1 / 2;

Líneas a la inversa

para poner un elemento al final de todo sin saber las columnas qeu hay, se puede poner la linea a la inversa, es decir, en vez de contar de izquierda a derecha, se cuenta de derecha a izquierda, poniendo números negativos.

no se usa start y end como en las lineas normales, sino que se pone desde donde se empieza / donde se acaba

      
        .item:nth-child(1){
          grid-row: 3 / 1;
          grid-column: 2 / 1;
        }
      
      El primer hijo (.item:nth-child(1)) se coloca en la tercera fila (de la línea 3 a la 1)

Líneas implícitas

las lineas implícitas son aquellas que se crean automáticamente cuando se añaden elementos al grid sin especificar su posición.

el grid se ajusta automáticamente para acomodar estos elementos, creando nuevas filas o columnas según sea necesario.

      El primer hijo (.item:nth-child(1)) se coloca en la segunda fila (de la línea 2 a la 3) y en la primera columna (de la línea 1 a la 2).
      
        .item:nth-child(1){
          grid-row: 2;
          grid-column: 1;
        }

Expansión de elementos

para que un elemento ocupe varias filas o columnas, se usa la palabra span

      El primer hijo (.item:nth-child(1)) se expande para ocupar dos filas (de la línea 1 a la línea 3) y se coloca en la primera columna (de la línea 1 a la línea 2).
      
        .item:nth-child(1){
          grid-row: 1 / span 2;
          grid-column: 1;
        }

Líneas negativas

las lineas negativas son las mismas que las normales, pero se cuentan de derecha a izquierda

      El primer hijo (.item:nth-child(1)) se coloca en la tercera fila (de la línea -2 a la -4) y en la segunda columna (de la línea -3 a la -4).
      
        .item:nth-child(1){
          grid-row: -2 / -4;
          grid-column: -3 / -4;
        }

Nombres de líneas

se pueden poner nombres a las lineas, para luego llamarlas desde los hijos

      El primer hijo (.item:nth-child(1)) se coloca en la fila que empieza en "inicio-filas" y termina en "inicio-fila2", y en la columna que empieza en "inicio-columnas" y termina en "inicio-columna3".
      
        .item:nth-child(1){
          grid-row: inicio-filas / inicio-fila2;
          grid-column: inicio-columnas / inicio-columna3;
        }
      
    
      Padre
      
        .grid-container{
          display: grid;
          grid-template-columns:
            [inicio-columnas] 1fr
            [inicio-columna2] 1fr
            [inicio-columna3] 1fr
            [fin-columnas];
          grid-template-rows:
            [inicio-filas] auto
            [inicio-fila2] auto
            [inicio-fila3] auto
            [fin-filas];
        }
      
      Hijos
      
        .item:nth-child(1){
          grid-row: inicio-filas / inicio-fila2;
          grid-column: inicio-columnas / inicio-columna3;
        }

Notas MielmaDev

Cuando es necesario, envolver en div con la clase oportuna, si hay varios elementos que comparten estilos similares, aprovechare la className.

max-width es aconsejable usar para que no haya problema en adaptar el diseño a diferentes tamaños de pantalla.

Si se usan porcentajes, el padre tiene que tener un ancho definido.

Si se usan unidades relativas (%, fr, auto), el diseño será más fluido y adaptable a diferentes tamaños de pantalla.

Puedes usar grid dentro de flex y a la inversa.

flex si necesitas solamente en un eje, en el horizontal sería la mejor opción

grid si necesitas en los dos ejes, horizontal y vertical.

Si tienes que hacer un diseño complejo, con muchos elementos, es mejor usar grid, si es un diseño sencillo, con pocos elementos, es mejor usar flex.

display: grid; llama a grid-template-columns y grid-template-rows para definir el contenedor grid, tanto sus columnas como las líneas.

Si no defines las filas, se crean automáticamente según el contenido.

Si no defines las columnas, se crea una única columna que ocupa todo el ancho del contenedor.

Si no defines ni filas ni columnas, se crea una única celda que ocupa todo el contenedor.

Si no defines ni filas ni columnas, pero tienes varios elementos hijos, se crean automáticamente filas para acomodar los elementos, todos en una única columna.

Con grid-template-columns y grid-template-rows defines explícitamente el número y tamaño de las columnas y filas. usa fr para un diseño más flexible. o repeat() para crear columnas o filas de igual tamaño. ejemplo: grid-template-columns: repeat(3, 1fr); repetir tres columnas de un fr.

grid-row y grid-column se utilizan para colocar los elementos hijos en el grid. ejemplo de grid-row: 1 / 2; y grid-column: 1 / 3;, marca desde la línea 1 hasta la línea 2 en el eje de las filas y desde la línea 1 hasta la línea 3 en el eje de las columnas.

Si no sabes cuántas filas o columnas habrá, puedes usar grid-auto-rows y grid-auto-columns para definir el tamaño de las filas y columnas que se creen automáticamente.

No se usa div extra, ya qeu se llama al padre, pero si hubiera muchos elementos seria buena práctica envolverlos en un div solamente lo que quieras qeu se envuelva, no tiene porque ser la section completa.

responsive puedes decirle que depende del tamaño de la pantalla sean mas columnas o menos.

gap se utiliza para definir el espacio entre las columnas y filas del grid.

Si usas grid-template-rows y grid-template-columns para definir el tamaño de las filas y columnas, el tamaño de las filas y columnas será fijo, si usas grid-auto-rows y grid-auto-columns, el tamaño de las filas y columnas será automático, dependiendo del contenido.

inline se utiliza para definir un elemento como un contenedor en línea, permitiendo que otros elementos fluyan a su alrededor.

inline-block se utiliza para definir un elemento como un contenedor en línea, pero permite establecer un ancho y alto específicos.

diferencia inline e inline-block es que inline no permite establecer un ancho y alto, mientras que inline-block sí.

Se llama con display: inline; y display: inline-block; respectivamente.

Al usar grid o flex es muy importante usar fracciones fr para no tener problemas con el gap o con el margen,o con el padding, si usas % puede que no funcione como esperas, ya que si cambias cualquier valor, se cambia todo el diseño.

Ordenar elementos con Flex y Grid

En Flexbox, el orden de los elementos se controla principalmente con flex-direction (por ejemplo, row o column) y con la propiedad order en los hijos. Ejemplo:

        
    .flex-container {
      display: flex;
      flex-direction: row; /* o column */
    }
    .flex-item {
      order: 2; /* Cambia el orden de aparición */
    }

En Grid, el orden visual se controla con grid-row y grid-column para ubicar los elementos en la celda deseada, independientemente del orden en el HTML. Ejemplo:

        
    .grid-container {
      display: grid;
      grid-template-columns: repeat(3, 1fr);
    }
    .grid-item:nth-child(2) {
      grid-column: 3;
      grid-row: 1;
    }

Así, con Flexbox se usa order para cambiar la posición de los hijos, y con Grid se usan grid-row y grid-column para colocarlos en la celda que quieras.

En resumen, Flexbox es ideal para diseños lineales y Grid para diseños bidimensionales. Ambos permiten controlar el orden visual de los elementos, pero lo hacen de maneras diferentes.
