23. Animaciones

Las transformaciones básicas que hemos visto, sólo permiten animar un elemento con un determinado tipo de transición. Podemos añadir más complejidad, juntando varias transiciones en un mismo elemento, incluso interactuando varios elementos entre sí. Para ello crearemos nuestra propia animación o Keyframe.

Propiedades individuales

Para definir una animación necesitamos varias propiedades, que podemos utilizar de forma individual o bien abreviada.

¿Qué hace? Código usado Efecto visual
La animación se reproduce siempre en la misma dirección (normal). 
animation: normal-demo 1.5s linear infinite;
@keyframes normal-demo {
  0% { left:0; }
  100% { left:40px; }
}
La animación alterna la dirección en cada ciclo (alternate). 
animation: alternate-demo 1.5s linear infinite alternate;
@keyframes alternate-demo {
  0% { left:0; }
  100% { left:40px; }
}
La animación se reproduce siempre al revés (reverse). 
animation: reverse-demo 1.5s linear infinite reverse;
@keyframes reverse-demo {
  0% { left:40px; }
  100% { left:0; }
}
La animación alterna la dirección, empezando al revés (alternate-reverse). 
animation: alternate-reverse-demo 1.5s linear infinite alternate-reverse;
@keyframes alternate-reverse-demo {
  0% { left:40px; }
  100% { left:0; }
}

Nombre

La propiedad animation-name especifica el nombre que va a llevar nuestra animación y que se establecerá posteriormente en la regla @keyframes que describe los fotogramas de la animación: 

animation-name: NombreAnimacion;

Duración

La propiedad animation-duration indica la cantidad de tiempo que tarda la animación en completarse, es decir, su duración: 

animation-duration: 10s;

Dirección

La propiedad animation-direction indica si la animación debe retroceder hasta el fotograma de inicio al finalizar la secuencia o si debe comenzar desde el principio al llegar al final. Se pueden utilizar los siguientes valores:

  • normal: Cada vez que termina un ciclo, la animación se reinicia al estado inicial y comienza desde el principio.
  • alternate: Cuando la animación termina, invierte su dirección, es decir, se reproduce al revés.
  • reverse: Cada ciclo de la animación se reproduce al revés.
  • alternate-reverse: Es similar a alternate pero la animación se reproduce al revés. Es decir, la primera reproducción empieza en el final, se reproduce al revés hasta al inicio y la segunda reproducción será normal, la tercera otra vez del revés, etc...
animation-direction: normal;

Repetición

La propiedad animation-iteration-count indica el número de veces que se repetirá la animación. Podemos especificar tanto un número de veces como hacer que sea infinita:

  • número: El número de veces que se repite la animación. Por defecto es 1.
  • infinite: Utilizando esta palabra clave la animación se repetirá continuamente, sin fin. 
animation-iteration-count: 3;
animation-iteration-count: infinite;

Ritmo

La propiedad animation-timing-function indica el ritmo de la animación, es decir, como se muestran los fotogramas de la animación, estableciendo curvas de aceleración.

Los valores son los mismos que hemos visto en las Transiciones con la propiedad de Tipo de transición (linear, ease-in, ease-out...): 

animation-timing-function: linear;

Estado

La propiedad animation-play-state permite pausar y reanudar la secuencia de la animación. Es muy útil si, por ejemplo, queremos detener una animación al pasar por encima (en el estado :hover) y luego reanudarla:

  • running: la animación se está ejecutando.
  • paused: la animación está pausada.
animation-play-state: running;

Retención de estilos

La propiedad animation-fill-mode especifica qué valores tendrán las propiedades después de finalizar la animación (los de antes de ejecutarla, los del último fotograma de la animación o ambos):

  • none: la animación no aplicará los estilos antes ni después de su ejecución.
  • forwards: el objeto sobre el que se aplica la animación mantendrá los valores y estilos que le aplique el último fotograma de la ejecución de la animación.
  • backwards: la animación aplicará los valores definidos en el primer fotograma tan pronto como se aplique al objeto, y los retendrá durante el tiempo indicado en animation-delay. El primer fotograma dependerá del valor de animation-direction.
  • both: la animación seguirá las reglas de las opciones forwards y backwards, extendiendo las propiedades de la animación en ambas direcciones.

Retardo

La propiedad opcional animation-delay indica el tiempo de retardo entre el momento en que el elemento se carga y el comienzo de la secuencia de la animación: 

animation-delay: 2s;

Propiedades Resumidas

Podemos también escribir las propiedades resumidas. El orden en que pongamos cada uno de los valores no importa, a excepción de la duración y el retrardo. El primer tiempo será siempre el de la duración y el segundo tiempo, el del retardo: 

animation: MiAnimacion 5s 1s 2 alternate backwards;

Declaración @keyframes

Una vez definido el comportamiento de la animación tenemos que declarar la animación en sí misma, indicando los cambios de comportamiento en cada fotograma, etc: 


.btn-rainbow{
  background-color:#FFF;
  color:#000;
}

.btn-rainbow:hover, .btn-rainbow:focus{
  animation: 10s infinite arcoiris;
}

@keyframes arcoiris {
  0% {background-color: #FFFFFF; color:#000;}
  12% {background-color: #E7484F;}
  24% {background-color: #f68B1D;}
  36% {background-color: #e1e23b;}
  50% {background-color: #639b47;}
  64% {background-color: #00AAC3; color:#000;}
  77% {background-color: #272b66; color:#FFF;}
  90% {background-color: #c466c1}
  100% {background-color: #FFFFFF; color:#000;}
}

Ejemplo de animación en imágenes


img {
  animation-name: camino;
  animation-duration: 5s;
  animation-direction: alternate;
  animation-iteration-count: infinite;
  animation-timing-function: linear;
  animation-play-state: running;
  animation-fill-mode: forwards;
}

img:hover{
  animation-play-state: paused;
}

@keyframes camino {
  0% {
    margin-left:0%;
    transform: rotate(0deg);
  }
  40%{
    transform: rotate(0deg);
  }
  50%{
    transform: rotate(360deg);
  }
  100% {
    margin-left: calc(100% - 175px);
    transform: rotate(360deg);
  }
}
