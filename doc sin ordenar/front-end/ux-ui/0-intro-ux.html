Experiencia de usuario

Diferencia entre UX y UI

UX (User Experience, experiencia de usuario) se refiere a cómo se siente una persona al interactuar con un producto o servicio digital. UI (User Interface, interfaz de usuario) es la parte visual y los elementos con los que el usuario interactúa, como botones, menús y colores. UX es la experiencia completa; UI es solo una parte de esa experiencia.

Proceso básico de UX

  • Investigación: Conocer a los usuarios y sus necesidades.
  • Prototipado: Crear bocetos o prototipos de la solución.
  • Testeo: Probar con usuarios reales y recoger feedback.
  • Iteración: Mejorar el producto según los resultados.

Herramientas comunes de UX/UI

  • Figma, Sketch, Adobe XD: para diseñar interfaces y prototipos.
  • Papel y lápiz: para bocetos rápidos.
  • Google Forms o encuestas: para recopilar opiniones de usuarios.

¿Qué es la Experiencia de Usuario (UX)?

La Experiencia de Usuario (UX) es el conjunto de percepciones, emociones y reacciones que una persona experimenta al interactuar con un producto, servicio o sistema digital. No se trata solo de si algo funciona, sino de cómo se siente el usuario al usarlo.

Objetivo

El objetivo principal de la UX es crear productos útiles, fáciles de usar y agradables, que resuelvan necesidades reales y generen una experiencia positiva y memorable.

Es por ello que el diseño está centrado en el usuario.

Ejemplo: una aplicación móvil para pedir comida.

Si el usuario encuentra el plato fácilmente, el pago es rápido y recibe confirmaciones y mensajes claros, la experiencia será buena.

Si la navegación es confusa o no entiende cómo finalizar el pedido, o si se ha realizado correctamente, su experiencia será negativa, aunque la comida sea excelente. Probablemente no vuelva a usar la aplicación.

Beneficios

Implementar una buena UX tiene muchos beneficios:

  • Mejora la satisfacción del usuario.
  • Mayor facilidad de uso, aprendizaje más sencillo y uso consistente
  • Aumenta la fidelización de los usuarios.
  • Aumenta las tasas de conversión de ventas.
  • Optimiza el tiempo de desarrollo al identificar problemas temprano.
  • Reduce costos de soporte al minimizar errores de uso.
  • Mejora el SEO con sitios web más usables y rápidos.
  • Refuerza la imagen de marca.

Inconvenientes

Por el contrario, si ignoramos la UX o la implementamos de manera pobre, tendremos problemas:

  • Continua necesidad de soporte (alto porcentaje de error/tarea).
  • Aumento de las tasas de abandono.
  • Aumento de los costes de desarrollo.
  • Transmitir mala imagen de marca.

Áreas de la UX

La UX es un campo multidisciplinar que integra varias áreas de conocimiento y disciplinas, entre ellas:

  • Arquitectura de la Información
  • Usabilidad
  • Diseño de Interacción
  • Investigación con Usuarios
  • Diseño Visual / Interfaz de Usuario
  • Accesibilidad
  • Estrategia de Contenidos

Todas las áreas trabajan mano a mano para poner al usuario en el centro y crear productos digitales más útiles, eficientes y agradables.

UX para Web

Según Peter Morville, en el contexto de la Web hay 7 Facetas de la Experiencia del Usuario que tenemos que cumplir:

  • Útil (funcionales)
  • Usable (usabilidad)
  • Deseable (diseño)
  • Encontrable (arquitectura de información y SEO)
  • Accesible (accesibilidad)
  • Creíble (reputación on-line)
  • Valioso (valor añadido)

Debe ser útil, usable, deseable, encontrable, accesible, creíble y valioso.

La apariencia debe ser agradable y atractiva para el usuario.

Ser encontrable significa que el sitio debe aparecer en buscadores como Google y que los elementos importantes, como el formulario de contacto, sean fáciles de localizar para el usuario.

Debe ser accesible, en el sentido de que cualquier persona puede usar esa página web o servicio, incluidas las personas que tienen una discapacidad. Los invidentes tienen que poder acceder a la web. La accesibilidad no es solo para lo que tiene una discapacidad, puede ser para cualquier persona que lo necesite o lo quiera usar.

Debes ser creíble: es importante construir una buena reputación. Si el sitio tiene nombres dudosos, carece de aviso legal o política de privacidad, o muestra precios poco realistas, puede generar desconfianza en los usuarios y afectar negativamente la experiencia.

por regla general para cada cosa lo hace un usuario diferente, es decir un uno para usabilidad, uno para accesibilidad, uno para credibilidad y así sucesivamente.

Esto significa que no se puede abordar la experiencia del usuario de manera uniforme; cada aspecto debe ser considerado y evaluado desde la perspectiva del usuario específico que interactúa con el producto.

En resumen, la experiencia del usuario es un campo complejo y multifacético que requiere un enfoque cuidadoso y matizado para cada uno de sus componentes.

Áreas de UX

Arquitectura de la información

La arquitectura de la información se refiere a la organización y estructuración de la información en un sitio web o aplicación para facilitar la navegación y el acceso a los contenidos.

Las cosas tienen que estar donde deben estar, tiene su lógica. No sería lógico que las naranjas de zumo estén en la sección de congelados.

Usabilidad

La usabilidad se refiere a la facilidad con la que los usuarios pueden interactuar con un producto o servicio. Un diseño usable permite a los usuarios completar tareas de manera eficiente y satisfactoria.

Ejemplo: el higiene y su complicación.

Diseño de interacción

El diseño de interacción se centra en crear interfaces efectivas y agradables para los usuarios, considerando cómo interactúan con el producto y cómo se sienten al hacerlo.

Lo lógico es que en un ascensor los botones de los pisos estén dentro de la cabina, no en la parte del portal.

Al clickar en un botón, este debe responder de manera clara y predecible, proporcionando feedback inmediato al usuario.

Al hacer clic en un botón, este debe responder de manera clara y predecible, proporcionando feedback inmediato al usuario.

Tiene que haber respuesta para saber que está funcionando lo que el usuario está haciendo.

Investigación con usuarios

La investigación con usuarios implica recopilar datos y opiniones de los usuarios para comprender sus necesidades, comportamientos y preferencias. Esta información se utiliza para informar el diseño y mejorar la experiencia del usuario.

Hay que investigar qué tipo de usuarios van a estar y cuáles no, y según eso sabes cómo tienes que diseñar. No es lo mismo una app para niños, que para madres, o para ancianos.
